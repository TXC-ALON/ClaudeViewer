# SchematicViewer 技术总结

## 一、项目概述

SchematicViewer 是一个基于 C++/Qt5 的原理图/电路图查看器，采用三层架构设计：
- **Commands Layer**: JSON 格式命令解析
- **Objects Layer**: 逻辑对象管理 (Inst, Pin, Net)
- **LayoutInfo Layer**: 几何布局信息
- **GUIItems Layer**: Qt 图形渲染

---

## 二、核心类设计

### 2.1 对象层 (Model)

| 类名 | 文件 | 功能 |
|------|------|------|
| `Obj` | `model/obj.h` | 所有对象的基类，包含 id_ 和 type_ |
| `Inst` | `model/obj.h` | 实例，属性：name, moduleName, parentId, isTop |
| `Pin` | `model/obj.h` | 引脚，属性：name, parentInstId, direction, hierPinId(可选) |
| `ObjectManager` | `model/object_manager.h` | 统一管理所有对象，提供查询和CRUD操作 |

**设计要点：**
- `ObjectID` 是 `uint64_t` 的别名，保证全局唯一性
- 使用 `std::optional<ObjectID>` 表示可选引用（如 hierarchical pin）
- `PinDirection` 枚举：`INPUT`, `OUTPUT`, `INOUT`

### 2.2 布局层 (Layout)

| 类名 | 文件 | 功能 |
|------|------|------|
| `LayoutInfo` | `layout/layout_info.h` | 布局信息基类 |
| `InstLayout` | `layout/layout_info.h` | 实例布局：boundingBox, position, nameTextSize, moduleTextSize |
| `PinLayout` | `layout/layout_info.h` | 引脚布局：anchorPos, side, offset |
| `LayoutManager` | `layout/layout_manager.h` | 自动布局算法实现 |

**设计要点：**
- `InstLayout::getFullBoundingBox()` 包含文本和引脚外延区域
- `PIN_EXT_LENGTH = 20` 引脚外延长度
- 布局算法考虑文本尺寸，避免文字重叠

### 2.3 视图层 (Qt Graphics)

| 类名 | 文件 | 功能 |
|------|------|------|
| `SchematicScene` | `view/schematic_scene.h` | 管理所有图形项的 QGraphicsScene |
| `InstGraphicsItem` | `view/inst_graphics_item.h` | 绘制实例矩形框 |
| `PinGraphicsItem` | `view/pin_graphics_item.h` | 绘制引脚线段和连接标记 |
| `TextItem` | `view/text_item.h` | 通用文本标签（复用性强） |

**Z-Order 分层：**
```
Z=10: Inst 矩形框
Z=20: Pin 引脚
Z=25: Text 文本标签
```

**设计要点：**
- 文本独立于 Inst 图形项，便于定位和更新
- 标签放置在 inst 矩形外部：name 在上方，moduleName 在下方
- TOP 实例使用虚线边框，普通实例使用实线

## 三、核心功能实现

### 3.1 命令格式 (JSON)

```json
// 加载实例
{"cmd": "load", "obj_type": "inst", "id": 1, "name": "top", "module_name": "TOP", "parent_id": null}

// 加载引脚
{"cmd": "load", "obj_type": "pin", "id": 2, "name": "a", "parent_inst_id": 1, "direction": "input"}

// 更新属性
{"cmd": "attribute", "obj_type": "inst", "id": 1, "module_name": "TOP"}

// 标记（选中）
{"cmd": "mark", "obj_type": "inst", "id": 1, "mark": true}
```

### 3.2 层次结构支持

- **Nested Hierarchy**: 子实例直接绘制在父实例内部
- **Hierarchical Pin**: 通过 `hierPinId` 关联层次引脚（可选属性）
- TOP 实例的识别：通过 `parentId` 为空或等于自身判断

### 3.3 自动布局算法

```cpp
void LayoutManager::autoLayout() {
    // Step 1: 处理嵌套层次 - 子实例放在父 TOP 实例内部
    // Step 2: 非嵌套实例使用贪心算法避让布局
    //         使用 InstBox 结构计算包含文本和引脚的完整包围盒
    //         碰撞检测使用 intersects() 方法
}
```

**布局参数：**
- `MIN_SPACING_X = 120` 最小水平间距
- `MIN_SPACING_Y = 100` 最小垂直间距

### 3.4 引脚视觉表示

- **0 连接**: 红色圆圈
- **1 连接**: 绿色菱形
- **>1 连接**: 橙色菱形

## 六、注意事项

1. **文本尺寸必须在布局时计算**: `TextItem::getTextSize()` 需要在设置字体后调用
2. **Z-Order 重要性**: 错误的 Z 值会导致标签被实例遮挡或引脚不可见
3. **引脚位置计算**: 需要父实例的包围盒信息来确定锚点
4. **嵌套层次处理**: 必须先处理 TOP 实例，再处理其子实例
5. **Qt 内存管理**: 使用 `std::unique_ptr` 管理图形项，添加场景时转移所有权
